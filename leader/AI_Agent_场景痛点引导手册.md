# AI Agent 场景引导与痛点梳理手册 (全研发生命周期)

> **使用说明**：
> 本手册用于在“落地探讨”环节，为不同角色的同学提供具体的思维发散方向。
> 管理者可直接引用其中的“痛点示例”来启发大家，或者把这份清单打印出来贴在白板上作为“灵感菜单”。

---

## 一、 产品/设计视角 (Product & Design)
**核心痛点**：文档维护难、需求传递失真、竞品分析耗时。

| 场景阶段 | 痛点描述 (Pain Points) | Agent 解决方案 (Reference Ideas) | 引导话术 |
| :--- | :--- | :--- | :--- |
| **需求分析** | 竞品调研要翻几十个网页，整理成表格太累。 | **竞品情报 Agent**：自动爬取竞品官网/App Store评论，总结功能亮点与槽点，生成对比表格。 | “如果你想知道竞品最近上了什么新功能，你希望 Agent 直接给你发一份周报吗？” |
| **PRD 撰写** | 需求变更频繁，PRD 改了这处漏了那处，开发总吐槽文档不一致。 | **PRD 质检 Agent**：输入新需求，自动扫描旧文档，标记出所有受影响的模块和潜在冲突点。 | “每次改需求，你最怕漏改哪个地方？如果 Agent 能帮你自动 Diff 出所有关联文档呢？” |
| **原型设计** | 简单的后台增删改查页面，画原型图纯属浪费时间。 | **原型生成 Agent**：输入“一个用户管理列表，要能筛选状态和导出”，直接生成低保真线框图或 HTML 代码。 | “那些千篇一律的后台列表页，你还想亲手画吗？能不能一句话生成？” |

## 二、 前端开发视角 (Frontend)
**核心痛点**：重复切图、接口联调扯皮、多端适配繁琐。

| 场景阶段 | 痛点描述 (Pain Points) | Agent 解决方案 (Reference Ideas) | 引导话术 |
| :--- | :--- | :--- | :--- |
| **编码开发** | 拿到设计稿，要把颜色、字体、间距一个个抄到 CSS 里，枯燥且容易错。 | **UI2Code Agent**：识别 Figma 设计稿，自动生成 React/Vue 组件代码，并提取 Tailwind 类名。 | “写 CSS 样式是不是最没技术含量的事？如果 Figma 直接吐出能用的代码，你愿意花多少时间修补？” |
| **接口联调** | 后端接口文档还没出，或者字段老变，Mock 数据写得手断。 | **智能 Mock Agent**：根据 Swagger/YAPI 文档，自动生成包含各种边界情况（空值、超长、异常）的 Mock 数据。 | “后端还没给接口，你希望 Agent 能帮你自动造出‘最像真的’假数据吗？” |
| **代码维护** | 接手老项目，全是 `any`，看不懂逻辑，不敢乱动。 | **代码考古 Agent**：选中一段复杂的遗留代码，解释其业务逻辑，并自动补充 TS 类型定义和注释。 | “面对那坨祖传代码，你最希望 Agent 告诉你什么？是‘这段代码在干嘛’，还是‘删了它会炸哪里’？” |

## 三、 后端开发视角 (Backend)
**核心痛点**：CRUD 重复劳动、SQL 优化难、日志排查累。

| 场景阶段 | 痛点描述 (Pain Points) | Agent 解决方案 (Reference Ideas) | 引导话术 |
| :--- | :--- | :--- | :--- |
| **API 开发** | 写 Controller、Service、Dao 层代码，大部分都是复制粘贴改名。 | **CRUD 生成 Agent**：根据数据库表结构，一键生成全套 RESTful API 代码及 Swagger 文档。 | “写增删改查是不是写吐了？如果 Agent 能把这一套全包圆了，你只写核心业务逻辑行不行？” |
| **数据库** | 慢 SQL 报警了，explain 看了半天不知道索引怎么加最合适。 | **DBA 助手 Agent**：分析慢查询日志，模拟执行计划，给出具体的索引添加建议或 SQL 重写方案。 | “SQL 跑得慢，你希望 Agent 直接把 `ALTER TABLE` 语句发给你，还是教你优化思路？” |
| **单元测试** | 业务逻辑复杂，覆盖率达不到要求，补单测简直是坐牢。 | **单测补全 Agent**：扫描代码中的分支逻辑，自动生成覆盖所有 if/else 路径的 JUnit/PyTest 测试用例。 | “为了凑代码覆盖率写的那些单测，能不能让 Agent 代劳？你只负责 Review？” |

## 四、 测试/QA 视角 (QA & Test)
**核心痛点**：用例编写量大、回归测试漏测、Bug 复现难。

| 场景阶段 | 痛点描述 (Pain Points) | Agent 解决方案 (Reference Ideas) | 引导话术 |
| :--- | :--- | :--- | :--- |
| **用例设计** | 看着 PRD 写测试用例，很容易漏掉异常场景（如断网、并发）。 | **边界测试 Agent**：分析需求文档，自动脑补出 50 种“破坏性”测试场景（如输入 Emoji、SQL 注入）。 | “你能不能让 Agent 扮演一个‘捣乱的用户’，专门帮你想怎么把系统搞崩？” |
| **UI 自动化** | 页面元素 ID 变了，自动化脚本全挂，维护脚本比测试还累。 | **自愈合测试 Agent**：当定位不到元素时，通过图像识别或语义分析自动寻找替代元素，并修复脚本。 | “自动化脚本老是挂，如果 Agent 能自己修脚本，你会不会觉得它是亲人？” |
| **Bug 归因** | 开发说“我本地是好的”，复现路径不明，来回扯皮。 | **Bug 猎手 Agent**：根据用户反馈的只言片语，自动在测试环境中尝试复现步骤，并录制视频。 | “用户报了个 Bug 但说不清怎么操作的，你希望 Agent 帮你试错复现吗？” |

---

## 五、 运维/DevOps 视角 (Ops)
**核心痛点**：告警轰炸、环境差异坑、发布担惊受怕。

| 场景阶段 | 痛点描述 (Pain Points) | Agent 解决方案 (Reference Ideas) | 引导话术 |
| :--- | :--- | :--- | :--- |
| **线上监控** | 手机一晚上响 100 次告警，大部分是误报或抖动，真故障被淹没了。 | **智能告警 Agent**：聚合相关联的告警，分析根因（Root Cause），只推送“经过确认的真故障”及处理建议。 | “半夜手机响了，你希望看到‘CPU 高’的报警，还是希望看到‘某服务死锁，建议重启’的诊断？” |
| **环境部署** | 开发环境好好的，一上预发就挂，配置差异查半天。 | **环境一致性 Agent**：自动对比不同环境的 ConfigMap、Nginx 配置、依赖版本，高亮显示差异项。 | “每次上线前，你最怕哪个配置没同步？如果 Agent 能帮你一键 Diff 所有环境配置呢？” |
| **故障恢复** | 线上出问题，手忙脚乱找文档、找负责人，错过最佳止损时间。 | **OnCall 助手 Agent**：检测到故障后，自动拉群、艾特相关负责人、抛出对应的 Runbook（操作手册）甚至自动执行回滚。 | “故障发生的那一刻，你最希望 Agent 帮你干什么？是帮你喊人，还是直接帮你回滚？” |
