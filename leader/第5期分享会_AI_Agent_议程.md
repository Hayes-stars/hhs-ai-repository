# 第5期分享会议程：从 0 到 1 打造 AI Agent (智能文件助手案例)

**会议主题**：手把手教你搭建“会思考”的 AI Agent & 探索业务落地场景
**会议时长**：60 分钟
**参与人员**：前端团队全员（含氛围官、质量把控官、项目负责人）
**核心目标**：
1.  **硬核拆解**：理解 ReAct 范式，掌握用 LangChain 搭建 Agent 的核心逻辑。
2.  **落地探讨**：共创 AI Agent 在现有工作/生活中的应用场景，并形成可落地的行动项。

---

## 一、 角色分工与职责 (Role Play)

| 角色 | 职责描述 | 关键动作 (Key Actions) |
| :--- | :--- | :--- |
| **氛围官** | 活跃气氛，激发脑洞 | 1. **控场**：在技术讲解环节若有冷场，及时抛出“小白问题”互动。<br>2. **引导**：在共创环节鼓励“疯狂的想法”，哪怕不切实际。 |

| **项目负责人** | 聚焦业务价值与落地 | 1. **关联**：不断发问“这个场景能复用到我们的系统里吗？”<br>2. **收敛**：在共创环节筛选出高价值、低成本的落地 idea。 |

## 二、 详细议程 (Agenda)

### 1. 硬核拆解：手把手教你搭建 AI Agent (35 分钟) —— [分享者]

*   **Part 1: 什么是“会思考”的 Agent？(5 min)**
    *   **核心概念**：大脑 (LLM) + 感知 (Tools) + 行动 (Action)。
    *   **为什么不是简单的 ChatGPT 对话？** 解析 ReAct (Reason + Act) 循环逻辑。

*   **Part 2: 实战演示 - 智能文件助手 (20 min)**
    *   **场景还原**：一个能够通过自然语言指令（如“帮我把桌面上的所有图片整理到 Photos 文件夹并重命名”）来管理文件系统的助手。
    *   **代码解构**：
        *   **Tools 定义**：如何封装文件操作 API（读取目录、移动文件、创建文件夹）给 AI 使用？
        *   **Todo List 机制**：Agent 如何拆解复杂指令为任务列表（Step 1: List files -> Step 2: Filter images -> Step 3: Move files）并逐一执行。
    *   **运行展示**：*现场演示 Agent 面对模糊指令时的自我修正与执行过程，展示 Thinking Log。*

*   **Part 3: 前端开发者的切入点 (10 min)**
    *   **LangChain.js**：在 Node.js 环境下实现文件操作 Agent 的差异点。
    *   **交互挑战**：当 Agent 执行高风险操作（如删除文件）时，前端如何设计 **Human-in-the-loop（人工确认）** 交互？

### 2. 落地探讨：AI Agent 场景共创 (20 分钟) —— [全员参与]

**目标**：不仅仅是“聊聊”，而是要沉淀出**具体的、可执行的**场景清单。

#### 环节设计：痛点 -> 场景 -> 方案 (3 步走)

*   **Step 1: 痛点大吐槽 (5 min)**
    *   **规则**：每人写出 1 个工作或生活中“最繁琐、重复、不仅费时还容易出错”的流程。
    *   *示例：每次发版都要手动去查 Jira 状态写更新日志；每次新人入职都要重复回答 Wi-Fi 密码和环境配置问题。*
    *   **工具**：便利贴 / 在线白板。

*   **Step 2: Agent 化改造 (10 min)**
    *   **规则**：分组讨论（3-4人一组），挑选 1 个最痛的痛点，设计一个 Agent 解决方案。
    *   **产出模板 (一句话 PRD)**：
        *   **Agent 名字**：(e.g., 周报杀手)
        *   **输入**：(e.g., 本周的 Git 提交记录 + Jira 任务列表)
        *   **核心技能 (Tools)**：(e.g., 访问 Git API, 读取 Jira, 调用 GPT-4 总结)
        *   **预期产出**：(e.g., 一份格式完美的 Markdown 周报)

*   **Step 3: 价值评估与收敛 (5 min) —— [项目负责人主导]**
    *   **投票**：大家对提出的 Agent 方案进行投票（维度：想用指数 & 实现难度）。
    *   **沉淀**：选出 Top 2 方案，列入“技术探索池”。

### 3. 总结与下一步 (5 分钟) —— [管理者]

*   **回顾**：总结 LangChain 的核心用法与 Agent 的设计思维。
*   **Action Item**：
    *   **文档沉淀**：将共创环节的 Top 2 方案整理到团队 Wiki。
    *   **PoC 认领**：鼓励感兴趣的同学认领其中一个方案，在两周后的分享会上展示 Demo。
