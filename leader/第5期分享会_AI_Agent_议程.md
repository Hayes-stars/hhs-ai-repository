# 第5期分享会议程：AI Agent 设计与前端交互创新 (HR 招聘助手案例)

**会议主题**：解构 AI Agent —— 从后端逻辑到前端体验（以 HR 招聘助手为例）  
**会议时长**：60 分钟  
**参与人员**：前端团队全员（含氛围官、质量把控官、项目负责人）  
**会议目标**：
1.  理解 AI Agent (ReAct 范式) 的核心运作原理。
2.  探讨 AI Agent 在前端交互（UI/UX）上的挑战与机会。
3.  共创 AI 在现有业务系统中的落地场景。

---

## 一、 角色分工与职责 (Role Play)

| 角色 | 职责描述 | 关键动作 (Key Actions) |
| :--- | :--- | :--- |
| **氛围官** | 活跃气氛，激发灵感，确保无人“掉线” | 1. **破冰**：主持开场的“画出 Agent”小游戏。<br>2. **控场**：在技术讲解过于枯燥时，抛出小白视角的问题或笑话调节气氛。 |
| **质量把控官** | 确保技术内容的深度与准确性 | 1. **追问**：针对 ReAct 模式的性能、延迟、错误处理提出挑战性问题。<br>2. **纠偏**：如果分享过于偏向 Python 后端，引导话题回到前端架构（如 LangChain.js）。 |
| **项目负责人** | 关注业务价值与落地可行性 | 1. **关联**：不断询问“这个功能能用在我们哪个项目里？”<br>2. **成本**：评估引入 Agent 的开发成本和 Token 消耗。 |

## 二、 详细议程 (Agenda)

### 1. 暖场与破冰 (5 分钟) —— [氛围官主导]
*   **活动**：“Agent 印象画”
    *   **规则**：每人用 30 秒在白板/纸上画出你心目中的“AI Agent”长什么样（是机器人？是大脑？还是一个黑盒？）。
    *   **目的**：暴露大家对 Agent 的初始认知差异，为后续讲解做铺垫。

### 2. 核心分享：HR Agent 设计实战 (30 分钟) —— [分享者]
*   **Part 1: 概念对齐 (5 min)**
    *   什么是 Agent？(Brain + Perception + Action)
    *   为什么是 LangChain？(简述 Python vs JS 版本差异，强调逻辑通用性)
*   **Part 2: 案例拆解 - HR 招聘助手 (15 min)**
    *   **业务场景**：自动筛选简历、发送邮件。
    *   **核心逻辑**：ReAct (Reason + Act) 循环演示。
    *   **代码走查**：重点讲解 Prompt Template 的构建和 Tool 的定义。
    *   *（建议调整点：此处需展示 Agent 思考过程中的日志输出，让前端看到“数据流”是如何变化的）*
*   **Part 3: 前端视角补充 (10 min)**
    *   **交互挑战**：当 Agent 思考 10 秒时，前端界面该怎么画？（Loading vs Streaming Steps）
    *   **Human-in-the-loop**：人工如何干预 Agent 的决策？

### 3. 深度问答 (15 分钟) —— [质量把控官/项目负责人引导]
*此环节由预设角色抛出问题，引导全员讨论：*

*   **Q1 (质量把控)**：“Agent 的 ReAct 模式通常需要多次 LLM 交互，响应很慢。前端除了转圈圈，有没有更好的**交互模式**来缓解用户焦虑？”
    *   *引导方向：流式输出 (Streaming UI)、骨架屏、步骤条可视化。*
*   **Q2 (项目负责)**：“我们的后台管理系统中，有哪些繁琐的表单填写或配置工作，是可以被这种 Agent 模式**替代**的？”
    *   *引导方向：一键导出复杂报表、自然语言配置活动规则。*
*   **Q3 (技术落地)**：“如果在浏览器端直接跑 LangChain.js，会有 Key 泄露风险吗？最佳的**前后端架构**应该是怎样的？”
    *   *引导方向：BFF 层转发、Edge Functions。*

### 4. 脑暴共创：Agent 的“UI 语言” (10 分钟) —— [全员参与]
*   **话题**：**"当 Agent 犯错时，前端怎么救？"**
*   **背景**：HR Agent 可能会幻觉，推荐错误的候选人。
*   **任务**：
    *   大家分组讨论，在白板上画出一个**“纠错 UI”**草图。
    *   是提供“重新生成”按钮？还是允许用户“手动修改中间步骤”？还是设计“置信度评分”？
*   **产出**：选出 1-2 个最佳交互方案，作为团队未来的 UI 规范参考。

### 5. 总结与行动 (5 分钟) —— [管理者]
*   **回顾**：总结 Agent 的核心价值（自动化复杂任务）与前端的核心价值（构建信任与体验）。
*   **Action Item**：
    *   分享者整理 LangChain.js 的相关文档/Demo 链接发到群里。
    *   感兴趣的同学认领一个“微型 Agent”课题（如：自动写周报 Agent），在下次分享会展示。
